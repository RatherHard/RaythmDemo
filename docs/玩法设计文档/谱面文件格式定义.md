## meta

歌曲元数据，存储了关于歌曲和谱面的基本信息，其键值为一个对象，记录了歌曲名称(title)、曲师名称(artist)、谱师名称(creator)、歌曲文件路径(path)等信息

## time

time 存储了歌曲中的 bpm 变化，bpm 与谱面流速变速有关，其键值为一个对象列表，例如：

```
time:[
    {
        beat:[0, 0, 1],
        bpm:120.0
    },
    {
        beat:[233, 0, 4],
        bpm:150.0
    }
]
```

beat: 整数型列表， bpm 变化所应用的位置。实际上，所有关于谱面位置（节拍）的参数都使用 beat 进行描述。 beat 列表中有三个数值，将其设为，代表第 x 小节、将每小节切分为 z 段后、第 y 段的位置，即拍
补充：一个小节为基准 bpm 下的四拍
bpm: 双精度浮点型，即新应用的 bpm 值

注：基准 bpm 为所有 bpm 中最接近所有 bpm 按时长加权平均得到的值的一个 bpm

## note

note 存储了谱面中的音符，其键值为一个对象列表，例如：

```
note:[
	{
		beat:[4, 0, 4],
		column:3
	},
	{
		beat:[4, 2, 4],
		endbeat:[5, 0, 4],
		column:1
	}
]
```

beat: 整数型列表，音符所处的位置，参见 time: beat
endbeat: 整数型列表，若音符种类为 Hold ，则存在此键（没有用于描述音符种类的键，仅依靠是否有 endbeat 区分 Note 和 Hold ），用于描述音符末尾所处的位置，参见 time: beat
column: 整数型，描述音符所处的轨道，表示其处在从左到右第个轨道上，其键值应为区间内的整数
