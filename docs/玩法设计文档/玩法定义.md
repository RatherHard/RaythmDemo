# 玩法定义

本文档定义 RaythmDemo 首个最小可玩切片的核心玩法规则，用于后续 Game 模块的谱面解析、输入判定、计分逻辑与测试用例设计。

---

## 1. 基础玩法

- **玩法类型**：4K 下落式音乐节奏玩法。
- **轨道数量**：4 条轨道。
- **音符类型**：
  - Note
  - Hold
- **判定基准**：以音频播放时间为主时钟，对比玩家输入时间与音符目标时间。

---

## 2. 音符定义

### 2.1 Note

Note 只有一个判定点，判定点即音符自身的目标时间。

| 属性 | 定义 |
|------|------|
| 判定点数量 | 1 |
| 物量 | 1 |
| 判定依据 | 玩家按下时间与音符目标时间的时间差 |

### 2.2 Hold

Hold 由头判和持续按压段组成，没有尾判。

| 属性 | 定义 |
|------|------|
| 头判数量 | 1 |
| 持续段数量 | 1 |
| 尾判 | 无 |
| 总物量 | 2 |

Hold 的 2 个物量分别来自：

1. **头判物量**：与 Note 使用相同判定区间，物量为 1。
2. **持续段物量**：根据玩家实际按住时长占长按总时长的比例进行判定，物量为 1。

---

## 3. 判定规则

### 3.1 Note 与 Hold 头判

时间差定义：

```text
时间差 = 玩家按下时间 - 音符目标时间
```

- 时间差为负数：提前按下。
- 时间差为正数：延后按下。

| 判定结果 | 判定区间 |
|----------|----------|
| Critical Perfect | `-20ms <= 时间差 <= +20ms` |
| Normal Perfect | `-40ms <= 时间差 <= +40ms`，且不属于 Critical Perfect |
| Great | `-80ms <= 时间差 <= +80ms`，且不属于 Perfect 区间 |
| Bad | `-120ms <= 时间差 < -80ms` |
| Miss | 其余情况 |

### 3.2 Hold 持续段

长按持续比例定义：

```text
持续比例 = 玩家实际按住时长 / 长按 Tap 总时长
```

| 判定结果 | 判定条件 |
|----------|----------|
| Critical Perfect | `持续比例 >= 80%` |
| Great | `持续比例 >= 60%`，且 `< 80%` |
| Miss | `持续比例 < 60%` |

> 说明：长按持续段不产生 Normal Perfect 或 Bad。

---

## 4. 物量规则

物量用于计分分摊与总分上限计算。

| 音符类型 | 物量计算 |
|----------|----------|
| Note | 1 |
| Hold | 2：头判 1 + 持续段 1 |

总物量定义：

```text
总物量 = 所有 Note 物量之和 + 所有 Hold 物量之和
```

---

## 5. 得分规则

### 5.1 总分上限

满分定义为：

```text
满分 = 1,000,000 + 总物量
```

每个物量的 Critical Perfect 满分为：

```text
Critical Perfect 单物量得分 = 1,000,000 / 总物量 + 1
```

### 5.2 单物量得分

| 判定结果 | 单物量得分 |
|----------|------------|
| Critical Perfect | `1,000,000 / 总物量 + 1` |
| Normal Perfect | `1,000,000 / 总物量` |
| Great | `500,000 / 总物量` |
| Bad | `0` |
| Miss | `0` |

### 5.3 总分计算

```text
总分 = 所有物量得分之和
```
