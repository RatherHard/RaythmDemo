# 游戏配置定义

本文档定义 RaythmDemo 当前玩法层所需的游戏配置草案。游戏配置用于描述玩家本地可调整的游玩参数，例如流速、轨道显示长度和判定偏移。

> 游戏配置与谱面文件不同：谱面文件描述歌曲、BPM 与音符数据；游戏配置描述玩家本地的显示和判定偏好。

---

## 1. 配置目标

游戏配置需要支持以下能力：

1. 调整谱面滚动速度。
2. 调整轨道显示长度。
3. 调整目押判定偏移。
4. 调整音押判定偏移。
5. 配置按键绑定。
6. 配置音量。
7. 配置窗口显示参数。
8. 明确谱面播放、歌曲播放、判定时机三者之间的相对关系。

---

## 2. 配置字段

| 字段 | 类型 | 范围 | 单位 | 说明 |
|------|------|------|------|------|
| `scrollSpeed` | number | `1.0` ~ `10.0` | 无 | 流速，最小调整单位为 `0.1` |
| `laneLength` | number | 待定 | 待定 | 轨道显示长度 |
| `visualOffset` | integer | 待定 | 毫秒 | 目押判定调整，谱面播放相对歌曲播放的偏移 |
| `audioOffset` | integer | 待定 | 毫秒 | 音押判定调整，歌曲播放相对判定时机的偏移 |
| `keyBindings` | object | 见下文 | 无 | 按键绑定配置 |
| `volume` | object | 见下文 | 无 | 音量配置 |
| `window` | object | 见下文 | 无 | 窗口显示配置 |

示例：

```json
{
    "scrollSpeed": 5.0,
    "laneLength": 1.0,
    "visualOffset": 0,
    "audioOffset": 0,
    "keyBindings": {
        "lane0": "KeyD",
        "lane1": "KeyF",
        "lane2": "KeyJ",
        "lane3": "KeyK"
    },
    "volume": {
        "master": 1.0,
        "music": 1.0,
        "effect": 1.0
    },
    "window": {
        "width": 1280,
        "height": 720,
        "fullscreen": false,
        "borderless": false,
        "vsync": true
    }
}
```

---

## 3. 流速 `scrollSpeed`

`scrollSpeed` 用于控制谱面音符在轨道上的滚动速度。

约束：

- 最小值为 `1.0`。
- 最大值为 `10.0`。
- 最小调整单位为 `0.1`。
- `scrollSpeed` 与实际屏幕移动速度的换算关系暂未定义，后续应结合基准 BPM、轨道长度和渲染坐标系统确定。

---

## 4. 轨道长度 `laneLength`

`laneLength` 用于控制轨道的显示长度。

当前约束暂未固化，后续需要明确：

- `laneLength` 的单位；
- 合法取值范围；
- 与渲染坐标、判定线位置和音符生成位置的关系；
- 是否影响实际判定，或仅影响视觉显示。

在未完成运行时设计前，`laneLength` 应仅作为显示配置，不应改变音符真实判定时机。

---

## 5. 目押判定调整 `visualOffset`

`visualOffset` 用于调整谱面播放相对歌曲播放的偏移，主要服务于玩家根据视觉位置进行判定的体验。

含义：

- 类型为整数。
- 单位为毫秒。
- 描述“谱面播放”相对“歌曲播放”的偏移量。

说明：

- 该字段用于调整玩家看到的音符位置与听到的音乐之间的对齐关系。
- 具体正负方向的运行时语义需要在 Game 判定实现时用测试固化。

---

## 6. 音押判定调整 `audioOffset`

`audioOffset` 用于调整歌曲播放相对判定时机的偏移，主要服务于玩家根据听觉节奏进行判定的体验。

含义：

- 类型为整数。
- 单位为毫秒。
- 描述“歌曲播放”相对“判定时机”的偏移量。

说明：

- 该字段用于调整玩家听到的音乐与实际判定时间之间的对齐关系。
- 具体正负方向的运行时语义需要在 Game 判定实现时用测试固化。

---

## 7. 按键绑定 `keyBindings`

`keyBindings` 用于描述玩法动作到键盘按键的映射。当前最小可玩切片优先覆盖 4K 轨道输入。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `lane0` | string | 是 | 第 0 轨对应按键 |
| `lane1` | string | 是 | 第 1 轨对应按键 |
| `lane2` | string | 是 | 第 2 轨对应按键 |
| `lane3` | string | 是 | 第 3 轨对应按键 |

示例：

```json
{
    "keyBindings": {
        "lane0": "KeyD",
        "lane1": "KeyF",
        "lane2": "KeyJ",
        "lane3": "KeyK"
    }
}
```

说明：

- 按键名称建议使用稳定的物理键位标识，而不是随键盘布局变化的字符。
- 当前阶段只定义 4K 默认轨道输入；暂停、返回、确认等菜单动作后续由设置系统扩展。
- 运行时应通过语义输入映射层将物理输入转换为 `LaneInputSnapshot`，Game 判定逻辑不应直接依赖具体键名。

---

## 8. 音量 `volume`

`volume` 用于描述本地音频播放音量。

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `master` | number | `0.0` ~ `1.0` | 总音量 |
| `music` | number | `0.0` ~ `1.0` | 音乐音量 |
| `effect` | number | `0.0` ~ `1.0` | 音效音量 |

示例：

```json
{
    "volume": {
        "master": 1.0,
        "music": 1.0,
        "effect": 1.0
    }
}
```

说明：

- `0.0` 表示静音，`1.0` 表示当前分类的最大配置音量。
- 实际输出音量可按 `master * music` 或 `master * effect` 计算。
- 当前 Audio 最小骨架尚未定义音效池；`effect` 先作为后续命中音、菜单音效等能力的预留配置。

---

## 9. 窗口 `window`

`window` 用于描述本地窗口显示参数。

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `width` | integer | `> 0` | 窗口宽度，单位为像素 |
| `height` | integer | `> 0` | 窗口高度，单位为像素 |
| `fullscreen` | boolean | `true` / `false` | 是否使用全屏模式 |
| `borderless` | boolean | `true` / `false` | 是否使用无边框窗口 |
| `vsync` | boolean | `true` / `false` | 是否启用垂直同步 |

示例：

```json
{
    "window": {
        "width": 1280,
        "height": 720,
        "fullscreen": false,
        "borderless": false,
        "vsync": true
    }
}
```

说明：

- `width` 与 `height` 描述窗口初始尺寸。
- `fullscreen` 与 `borderless` 同时为 `true` 时，运行时应明确选择一种优先级；当前建议优先解释为无边框全屏。
- `vsync` 会影响渲染提交节奏，但不应改变 Game 判定时间线。

---

## 10. 时间关系说明

游戏运行时至少存在三条相互独立但需要对齐的时间线：

| 时间线 | 说明 |
|--------|------|
| 谱面播放 | 音符在视觉轨道中的推进时间 |
| 歌曲播放 | 音频实际播放时间 |
| 判定时机 | 输入被判定时使用的逻辑时间 |

从当前配置可以看出：

- 谱面播放、歌曲播放、判定时机三者是相互独立的。
- `visualOffset` 影响谱面播放与歌曲播放之间的对齐。
- `audioOffset` 影响歌曲播放与判定时机之间的对齐。
- 后续实现应避免把三者强行合并成同一个时间值，而应在统一时间源基础上显式计算偏移后的时间。

---

## 11. 暂不定义的内容

为避免当前阶段过度设计，以下内容暂不在本配置中固化：

- 配置文件的实际存储路径。
- 配置版本迁移策略。
- 按键冲突处理、手柄输入、触摸输入和多键位绑定规则。
- 音量曲线、响度标准化、静音快捷键和分设备输出策略。
- 多显示器选择、窗口位置记忆、分辨率白名单和渲染质量档位。
- `scrollSpeed` 到实际像素速度或世界坐标速度的换算公式。
- `laneLength` 的最终单位和范围。
- `visualOffset` 与 `audioOffset` 正负号的最终运行时解释。

这些内容应在 Game 最小可玩切片或设置系统进入开发计划后再补充。
