## 2026-07-04

### 建立 Platform 事件泵并收缩窗口事件职责

**完成内容**

- 新增 `include/Platform/EventPump.hpp` 与 `src/Platform/EventPump.cpp`，建立薄 `Raythm::Platform::EventPump`，作为 SDL 全局事件队列中窗口事件的统一消费与翻译入口。
- 将窗口事件翻译逻辑从 `Window::pollEvent()` 迁移到 `EventPump::pollWindowEvent()`，由事件泵统一推进 SDL 队列，避免非窗口事件或其他窗口事件阻塞当前窗口生命周期事件。
- 收缩 `Raythm::Platform::Window` 职责，移除 SDL 队列轮询入口，新增 `applyEvent(const WindowEvent&)` 与 `getWindowId()`，使窗口对象主要负责 SDL 窗口资源、窗口 ID 与 sticky close 状态。
- 更新 `src/main.cpp` 最小主循环，由 `EventPump` 拉取窗口事件，先应用到 `Window`，再转发给 `Renderer::handleWindowEvent()`。
- 更新 `tests/window_test.cpp`，复用 `RaythmDemoWindowTest` 覆盖事件泵翻译、非窗口事件跳过、其他窗口事件跳过、unsupported owned window event 跳过、quit/close 关闭状态应用。
- 更新 `CMakeLists.txt`，将 `src/Platform/EventPump.cpp` 加入 `RaythmDemoPlatform`。

**验证结果**

- `cmake --build /e/RaythmDemo/build --config Debug` 构建通过。
- `ctest --test-dir /e/RaythmDemo/build -C Debug --output-on-failure` 执行通过，3/3 tests passed。
- `ctest --test-dir /e/RaythmDemo/build -C Debug -R Window --output-on-failure` 执行通过，1/1 tests passed。
- 已检查 `.clangd`，本次仅新增 `src/Platform/EventPump.cpp` 并复用既有 `include/` 搜索路径和平台宏，无需同步修改。

**后续计划/遗留问题**

- 后续继续定义输入事件类型，并在 `EventPump` 中扩展键盘、鼠标按键与鼠标移动事件翻译。
- 后续建立输入状态系统，维护 key down/up、mouse button down/up、mouse motion 与帧边沿状态。
- 当 `Core::Application` 主循环建立后，将当前 `main` 中显式事件消费与分发逻辑迁移到 Core 层。

## 2026-07-04

### 建立 Render Vulkan 清屏与 Present 骨架

**完成内容**

- 新增 `include/Render/Renderer.hpp` 与 `src/Render/Renderer.cpp`，建立 `Raythm::Render::Renderer` RAII 渲染器，负责 Vulkan instance、SDL surface、physical/logical device、swapchain、image view、command buffer、同步对象的生命周期。
- 渲染路径使用 `vkCmdClearColorImage` 对 swapchain image 执行最小清屏，并通过 present queue 稳定提交到 SDL Vulkan 窗口。
- 在 `Renderer::handleWindowEvent()` 与 `Renderer::renderFrame()` 中接入 resize/minimize/out-of-date/suboptimal 处理，drawable size 为 0 时暂停渲染，恢复后重建 swapchain。
- 修正 `Window::pollEvent()` 对 unsupported owned window event 与 `SDL_EVENT_QUIT` 的处理，避免事件队列卡住并让应用级退出请求能够进入主循环关闭路径。
- 将 `src/main.cpp` 从入口占位更新为最小 SDL 视频初始化、窗口创建、Renderer 创建、事件转发、清屏 Present 主循环。
- 在 `CMakeLists.txt` 中新增 `RaythmDemoRender` 与 `RaythmDemoRenderTest`，并将渲染测试注册到 CTest。

**验证结果**

- `cmake --build /e/RaythmDemo/build --config Debug` 构建通过，生成 `RaythmDemoRender`、`RaythmDemo` 与 `RaythmDemoRenderTest`。
- `ctest --test-dir /e/RaythmDemo/build -C Debug --output-on-failure` 执行通过，3/3 tests passed，包含新增 `RaythmDemoRenderTest`。

**后续计划/遗留问题**

- 若构建或 CTest 暴露 Vulkan 驱动、SDL Vulkan surface 或 MSVC 编译问题，需要按最小差异修复。
- 后续可继续拆分纯选择策略测试，例如 surface format、present mode、extent 与 queue family 选择，以降低 GPU 环境依赖。
