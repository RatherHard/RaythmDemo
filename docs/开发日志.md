- 新增 `include/Core/Application.hpp` 与 `src/Core/Application.cpp`，建立 `Raythm::Core::Application`，接管 SDL video 初始化、窗口创建、事件泵、输入状态、Renderer 初始化、主循环、退出请求和 Renderer idle/shutdown 顺序。
- 将 `src/main.cpp` 收缩为最小入口，仅创建 `Core::Application` 并调用 `run()`。

### 建立 Platform 输入状态系统

**完成内容**

- 新增 `include/Platform/InputState.hpp` 与 `src/Platform/InputState.cpp`，建立 `Raythm::Platform::InputState`，消费 `InputEvent` 并维护键盘、鼠标按键、鼠标位置与当前帧相对移动状态。
- `InputState` 支持 `beginFrame()` 清理按下/释放边沿与鼠标 delta，同时保留 held 状态；支持 `clear()` 重置 held、edge 与鼠标坐标。
- 键盘状态使用 `SDL_Scancode` 作为物理键索引，并通过范围检查避免越界；重复 key-down 保持 held 状态但不重新触发 pressed edge。
- 鼠标按键使用 SDL 1-based button id 规范化到固定容量数组；鼠标移动在同一帧内累计 `xRelative`/`yRelative`，绝对坐标保留最后一次位置。
- 更新 `src/main.cpp`，在每帧开始调用 `InputState::beginFrame()`，将统一事件流中的 input payload 分发到 `InputState`，并在 `FocusLost` 时调用 `InputState::clear()` 避免焦点切换后 held 输入卡住。
- 新增 `tests/input_state_test.cpp` 与 CTest 目标 `RaythmDemoInputStateTest`，覆盖默认空状态、key down/up、repeat 抑制、同帧 press/release、mouse button down/up、mouse position/delta 与 clear reset。
- 更新 `CMakeLists.txt`，将 `src/Platform/InputState.cpp` 加入 `RaythmDemoPlatform`，并注册 `RaythmDemoInputStateTest`。
